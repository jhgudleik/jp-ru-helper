
# JP-RU Historical Terms Helper

Интеллектуальный «исторический глоссарий» для переводчиков-японистов.  
Агент сам находит японский термин в тексте, вытягивает русскоязычную статью из Википедии и выдаёт готовую справку: рекомендованный перевод на основе заголовка параллельной русскоязычной статьи, дословный перевод и 1-2 предложения толкования термина.


## Демо

![Анимация работы градио-интерфейса](jp_ru_gradio.gif?v=2)

---


## Что умеет
- выделяет японский термин из произвольной фразы (кандзи / кана);
- по Викиданным ищет русскую «сестринскую» статью;
- генерирует краткую справку на русском с учётом Википедии;
- показывает метрики: время, кол-во LLM- и Wiki-запросов.


---

## Зачем вообще «агент» вместо простого словаря?

| Классический словарь | Агент JP-RU |
|----------------------|-------------|
| Фиксированный набор слов | Динамически находит **любой** термин, пока он есть в японской Википедии |
| Одна «каноническая» лемма | Учитывает **контекстные синонимы** и исторические переименования (萩藩 → 長州藩 → Тёсю) |
| Только перевод | Даёт **готовую статью-заготовку** для клиента: название, дословный перевод, 1-2 предложения фона |
| Ручной поиск русского источника | Автоматом тянет **русскую сестринскую статью** по Викиданным |
| Не видит, что **внутри** японского текста | Сам **извлекает** подозрительный термин из любой фразы пользователя |
| Нет обратной связи | Показывает **метрики**: сколько запросов ушло и как быстро ответил |

## Как это экономит время переводчика-япониста

1. **Ctrl+C → Ctrl+V** вместо цепочки «открыть японскую Википедию → найти QID → перейти в русскую → скопировать название → придумать пояснение».  
2. **Единое окно** в браузере: не нужен ни Deepl, ни Google-переводчик, ни закладки со словарями.  
3. **Сразу готовый текст для сноски/комментария заказчику** – можно вставлять прямо в Word или CAT-инструмент.  
4. **offline-совместимость**: модель 1.5B помещается на 4 ГБ VRAM, можно работать без интернета (если русская Вики уже кэширована).  
5. **Прозрачность**: видно, сколько LLM-вызовов сделано – легче оценить нагрузку и стоимость при масштабировании на всю редакцию.

## Быстрый старт

1. Установите зависимости  
   ```bash
   pip install -r requirements.txt
   # или вручную:
   pip install gradio transformers torch wikipedia requests
   ```

2. Запустите  
   ```bash
   python jp_ru_gradio.py
   ```
   Откроется браузерная страница `http://127.0.0.1:7860` (если порт занят, задайте `GRADIO_SERVER_PORT`).

3. Введите фразу, например:  
   `В тексте встретилось 江戸幕府. Как перевести?`  
   и получите готовую справку.

4. Кнопка **Flag** в интерфейсе Gradio позволяет пользователям сообщать о некорректных или неудачных ответах системы. При нажатии она сохраняет контекст запроса и ответа в логи, что помогает разработчикам анализировать ошибки и повышать качество перевода исторических терминов. Использование Flag превращает интерфейс из пассивного инструмента в активную систему обратной связи, способствующую постоянному обучению и адаптации помощника к реальным потребностям переводчиков-японистов.
